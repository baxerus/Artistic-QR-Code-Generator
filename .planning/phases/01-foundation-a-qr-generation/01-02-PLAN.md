**URL Validation (INPUT-01):**
```javascript
function isValidURL(string) {
  try {
    const url = new URL(string);
    return url.protocol === 'http:' || url.protocol === 'https:';
  } catch (err) {
    return false;
  }
}
```
Use native URL constructor -- NOT regex. Only accept http: and https: protocols.

**Version Capacity Table (INPUT-02):**
Hardcoded lookup table for QR error correction level H, byte mode:
```javascript
const CAPACITIES_H_BYTE = [
  { version: 1, capacity: 7 },
  { version: 2, capacity: 14 },
  { version: 3, capacity: 24 },
  { version: 4, capacity: 34 },
  { version: 5, capacity: 44 },
  { version: 6, capacity: 58 },
  { version: 7, capacity: 64 },

```

**Minimum Version Calculation (INPUT-02):**
```javascript
function calculateMinimumVersion(url) {
  const byteLength = new Blob([url]).size;
  for (const entry of CAPACITIES_H_BYTE) {
    if (byteLength <= entry.capacity) {
      return entry.version;
    }
  }
  return -1; // URL too long for version 8
}
```
Use `new Blob([url]).size` for accurate byte count (handles Unicode correctly).

**Version Dropdown Update (INPUT-03, INPUT-04):**
```javascript
function updateVersionDropdown(minVersion) {
  const dropdown = document.getElementById('version-select');
  dropdown.innerHTML = '';
  for (let v = minVersion; v <= 8; v++) {
    const option = document.createElement('option');
    option.value = v;
    option.textContent = 'Version ' + v + ' (' + getModuleSize(v) + '\u00D7' + getModuleSize(v) + ')';
    dropdown.appendChild(option);
  }
  // Auto-select minimum version
  dropdown.value = minVersion;
}

function getModuleSize(version) {
  return 17 + (version * 4); // QR spec: modules = 17 + 4*version
}
```
Only show versions >= minimum for entered URL. Per locked decision: restrict dropdown options only (no explicit text about "minimum version").

**QR Code Generation (QR-01):**
```javascript
function generateQR() {
  const url = document.getElementById('url-input').value.trim();
  if (!isValidURL(url)) return;

  const version = parseInt(document.getElementById('version-select').value);
  const canvas = document.getElementById('qr-canvas');

  QRCode.toCanvas(canvas, url, {
    errorCorrectionLevel: 'H',
    version: version,
    margin: 4,
    width: 400,
    color: { dark: '#000000', light: '#FFFFFF' }
  }, function(error) {
    if (error) {
      console.error('QR generation failed:', error);
    }
  });
}
```
Error correction level H is hardcoded per project decision. Margin 4 is required quiet zone. Width 400 provides good quality (CSS handles display scaling).

**Validation UI Updates:**
When URL input changes:
- Empty: remove all validation classes, disable generate button
- Valid URL: add `.valid` class, remove `.invalid`, enable generate button, update version dropdown
- Invalid URL: add `.invalid` class on blur, remove `.valid`, disable generate button
- URL too long for version 8: show invalid state (the URL won't fit even in our max version)

Validation icon behavior:
- Valid: show green checkmark (Unicode &#10003; or CSS ::after content)
- Invalid: show red X mark (Unicode &#10007; or CSS ::after content)
- Empty/neutral: hide icon

**Event Wiring:**
All within `DOMContentLoaded`:
1. URL input `input` event: validate URL, if valid update version dropdown + enable button, if empty clear state
2. URL input `blur` event: show invalid indicator if non-empty and invalid
3. Generate button `click` event: call generateQR()
4. URL input `keypress` event: if Enter key, call generateQR() (convenience)
5. Version dropdown `change` event: if URL is valid, regenerate QR with new version

**Generate button state:**
- Disabled by default (no URL entered yet)
- Enabled only when URL is valid and fits within version 8 capacity
- Visual disabled state: grayed out, cursor not-allowed

**IMPORTANT:** Do NOT use alert() for errors. Use console.error for debugging. Invalid states are communicated via the validation UI only (per locked decision: subtle approach).
