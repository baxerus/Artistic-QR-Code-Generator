Implementation of `initShiftLegendFeedback()`:

```javascript
function initShiftLegendFeedback() {
  document.addEventListener('keydown', function(e) {
    if (e.key === 'Shift') {
      const buttons = document.querySelectorAll('#paint-legend .legend-btn');
      buttons.forEach(btn => {
        if (btn.dataset.color === 'unset') {
          btn.classList.add('active');
        } else {
          btn.classList.remove('active');
        }
      });
    }
  });

  document.addEventListener('keyup', function(e) {
    if (e.key === 'Shift') {
      // Restore the actual selectedColor highlight
      const buttons = document.querySelectorAll('#paint-legend .legend-btn');
      buttons.forEach(btn => {
        if (btn.dataset.color === selectedColor) {
          btn.classList.add('active');
        } else {
          btn.classList.remove('active');
        }
      });
    }
  });
}
```

Per locked decisions:
- Legend highlights "unset" option when Shift is held — uses same `.active` CSS class (background: #bfdbfe, border-color: #3b82f6, font-weight: 500)
- Only "unset" appears selected while Shift held — original selection indicator hidden temporarily (other buttons lose `.active`)
- On Shift release, original `selectedColor` selection is restored
- No cursor change — user relies on key feel (per decision)

Find where `initLegendHandlers()` is called in the initialization flow and add `initShiftLegendFeedback()` call immediately after it.
