<task type="auto">
  <name>Task 2: Implement drag-to-move with grid snapping and wrap-around</name>
  <files>index.html</files>
  <action>
Add Move tool drag behavior inside setupPaintingEvents. When Move is active, pointerdown starts a move session (store start cell + snapshot of paintGrid cells), pointermove computes integer dx/dy in module units, and renders a live preview of the shifted pattern without committing to paintGrid. Pointerup commits the shift only if dx/dy != 0 (click-without-drag no-op).

Shifts must snap to whole modules, allow diagonal movement, and wrap around edges (toroidal). Move only painted pixels: construct a new grid from the snapshot and write moved pixels into a fresh array so moved pixels always win. On commit, update paintGrid.cells, re-render, scheduleDecodeUpdate(), resetOptimizationResults(), checkPaintState(), and savePattern(). Do not trigger decode updates during drag.
  </action>
  <verify>Paint a pattern, select Move Pattern, drag diagonally across edges, and confirm the pattern wraps and reappears on the opposite side after release.</verify>
  <done>Dragging with Move Pattern visibly shifts the entire painted pattern by whole modules with wrap-around, and release commits without changing pixel values.</done>
</task>
