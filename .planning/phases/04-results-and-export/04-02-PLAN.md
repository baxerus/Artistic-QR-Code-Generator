**New JS function** (add near the other result functions):

`renderErrorOverlay(overlayCanvas, moduleData, moduleCount, paintedPixels, functionMask)`:
- Parameters: overlayCanvas (canvas element), moduleData (array of 0/1 from result), moduleCount (int), paintedPixels (from paintGrid.cells), functionMask (from createFunctionPatternMask)
- Set overlayCanvas.width = moduleCount, overlayCanvas.height = moduleCount (matches QR thumbnail size, CSS scales to 80px with pixelated rendering)
- Get 2d context, clearRect the full canvas
- Iterate row 0..moduleCount-1, col 0..moduleCount-1:
  - idx = row * moduleCount + col
  - If functionMask[idx] === 1: fill with rgba(0, 100, 255, 0.35) — blue for function patterns
  - Else if paintedPixels[idx] !== 0 (pixel is locked):
    - paintedIsDark = (paintedPixels[idx] === 2)
    - qrIsDark = (moduleData[idx] === 1)
    - If paintedIsDark === qrIsDark: fill rgba(0, 200, 0, 0.3) — green for match
    - Else: fill rgba(255, 0, 0, 0.45) — red for conflict
  - Else: no overlay (unset data pixel)
- Use fillRect(col, row, 1, 1) since canvas is at module resolution (CSS scales it)

**Important implementation notes:**
- paintGrid is a global. Its `.cells` array has the paint states (0=unset, 1=white, 2=black)
- createFunctionPatternMask(version) is an existing global function returning an array

- When updateTopResults is called during live search, overlays should be cleared (they rebuild from new data on next toggle)
- The overlay is rendered lazily (only on toggle, not on every result update) to avoid performance issues during live search
