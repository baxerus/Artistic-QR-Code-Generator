Current sort logic:

topResults.sort((a, b) => {

  }

Modified sort logic:
```javascript
// Sort: decodable first, then by RS corrections (ascending), then by pixelDiff (ascending)
// Lower RS = more reliable QR. Lower pixelDiff = closer to painted art.
topResults.sort((a, b) => {
  // Decodable always beats non-decodable
  if (a.decodable !== b.decodable) {
    return b.decodable ? 1 : -1;
  }
  // Among decodable: prefer lower RS corrections (fewer corrections = more reliable)
  if (a.rsCorrections !== b.rsCorrections) {
    return a.rsCorrections - b.rsCorrections;
  }
  // Tiebreaker: prefer lower pixel diff (closer to art)
  return a.pixelDiff - b.pixelDiff;
});
```

Note: Phase 13 will redefine ranking to use RS as primary sort. For now, we just integrate RS into the existing sorting scheme where decodable is still the top-level filter.
