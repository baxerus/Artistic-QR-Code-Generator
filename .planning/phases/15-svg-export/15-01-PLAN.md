  <action>
Add two functions near the existing `downloadCanvasAsPNG` function (around line 16116):

**`generateSVG(moduleData, moduleCount)`** — Creates SVG string from QR module data:
- Add 4-module quiet zone: total viewBox is `(moduleCount + 8)` × `(moduleCount + 8)` (4 modules padding on each side)
- ViewBox: `0 0 {moduleCount+8} {moduleCount+8}` — 1 unit per module, infinitely scalable
- Omit explicit `width`/`height` attributes (viewBox only, per user decision)
- Include `xmlns="http://www.w3.org/2000/svg"` attribute
- Include `<title>` element containing the encoded URL (`currentURL + '#' + hash`) — pass URL as parameter
- White background `<rect>` filling entire viewBox: `x="0" y="0" width="{moduleCount+8}" height="{moduleCount+8}" fill="#FFF"`
- Single `<path>` element with `fill="#000"` for all black modules using M/h/v commands:
  - Iterate through moduleData row by row
  - For each black module at (col, row), emit `M{col+4} {row+4}h1v1h-1z` (offset by quiet zone)
  - Concatenate all path commands into one `d` attribute
- Include XML prologue `<?xml version="1.0" encoding="UTF-8"?>` for maximum compatibility
- Return the complete SVG string

**`downloadSVG(svgString, filename)`** — Downloads SVG string as file:
- Create Blob with type `image/svg+xml`
- Use same createObjectURL + anchor click + revokeObjectURL pattern as `downloadCanvasAsPNG`
- Filename: `${filename}.svg`
