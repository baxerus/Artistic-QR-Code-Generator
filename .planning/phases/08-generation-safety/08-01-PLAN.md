   function hasHashFragment(urlString) {

       return url.hash.length > 0;

       return false;

Add pattern existence check and guard the version change handler:

1. Create `hasPattern()` helper function (place near other helper functions, around line 13990):
   ```javascript
   function hasPattern() {
     if (!paintGrid) return false;
     for (let i = 0; i < paintGrid.cells.length; i++) {
       if (paintGrid.cells[i] !== 0) {
         return true;
       }
     }
     return false;
   }
   ```
   Note: This extracts the existing pattern from checkPaintState() into a reusable helper.

2. Update the version change event handler (around line 15919) to guard auto-generation:
   Replace the current handler:
   ```javascript
   versionSelect.addEventListener("change", function () {
     saveVersion(parseInt(this.value, 10));
     const url = urlInput.value.trim();
     if (isValidURL(url) && !generateBtn.disabled) {
       generateQR();
     }
   });
   ```
   
   With guarded version:
   ```javascript
   versionSelect.addEventListener("change", function () {
     const newVersion = parseInt(this.value, 10);
     saveVersion(newVersion);
     
     const url = urlInput.value.trim();
     if (!isValidURL(url) || generateBtn.disabled) return;
     
     // SAFE-02/03: If pattern exists, don't auto-generate
     // User must click Generate button explicitly
     if (hasPattern()) {
       return;
     }
     
     generateQR();
   });
   ```

3. Verify the Generate button click handler (around line 15907) continues to work normally - it should always call generateQR() regardless of pattern existence. This satisfies SAFE-03 (only explicit Generate triggers generation when pattern exists).

Note: SAFE-04 (pattern preservation on same-version) already works - initPaintCanvas() calls loadPattern() which restores the saved pattern. No changes needed.
