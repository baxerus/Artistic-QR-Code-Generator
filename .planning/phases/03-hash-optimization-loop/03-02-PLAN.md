**Part C: Button state machine**

The optimize button has three states, controlled by a variable `searchState` ('idle' | 'running' | 'completed'):

- **idle**: Button text "Generate", enabled only when paintGrid has at least one non-zero cell. Click calls `handleOptimizeClick()` which starts search.
- **running**: Button text "Stop", always enabled. Click calls `handleOptimizeClick()` which stops search.
- **completed**: Button text "Run again", enabled. Click calls `handleOptimizeClick()` which restarts search (accumulating results).

`handleOptimizeClick()` dispatches based on `searchState`:
- idle/completed -> start optimization: create worker (if not exists), post START_OPTIMIZATION with current state + existing top results (for accumulation), set searchState='running', show progress, hide summary
- running -> stop: post STOP_OPTIMIZATION, let COMPLETE handler transition state

**Part D: Enable/disable optimization button based on paint state**

After each paint action (in the pointerdown handler) and clear button:
- Check if paintGrid has any non-zero cells
- If yes: enable optimize button, hide paint-hint
- If no: disable optimize button, show paint-hint
- Also show optimization-section when QR is generated (in generateQR function)
- Also call this check in runCorruptionPipeline or after it

**Part E: Wire worker onmessage to UI updates**

Update the worker onmessage handler (created in Plan 01) to call UI update functions:

On PROGRESS message:
- `updateProgressBar(elapsed, timeout)` -- update progress bar width, time text
- `updateSearchStats(attempts, elapsed)` -- update attempts counter and elapsed text
- `updateTopResults(topResults)` -- render/update top 5 QR preview canvases

On COMPLETE message:
- Set `searchState = 'completed'`
- Update button text to "Run again"
- Hide progress bar
- Show summary line: "Done - {attempts} attempts, {decodableCount} decodable found"
- Final call to `updateTopResults(topResults)` with final results
- Store topResults in `currentTopResults` for accumulation on next run

**Part F: Top 5 rendering**

`updateTopResults(topResults)` function:
1. Show the top-results container
2. For each of up to 5 results:
   - Get or create the result slot DOM element
   - Create a canvas element sized to moduleCount x moduleCount
   - Draw the moduleData (Uint8Array of 0/1 values) onto the canvas -- 1 pixel per module, black for 1, white for 0
   - Set canvas CSS to 80x80px with `image-rendering: pixelated` for crisp scaling
   - Set label text: "{decodable ? 'decodable' : 'not decodable'} - {pixelDiff} errors"
   - If this result is NEW to the top 5 (wasn't in previous update), add highlight animation class
3. Remove any extra slots if fewer than 5 results
4. Results are already sorted by worker (decodable first, then by pixelDiff ascending)

**Part G: Highlight animation**

CSS keyframe animation for new best result entering top 5:
```css
@keyframes result-highlight {
  0% { box-shadow: 0 0 0 0 rgba(76, 175, 80, 0.7); }
  50% { box-shadow: 0 0 12px 4px rgba(76, 175, 80, 0.5); }
  100% { box-shadow: 0 0 0 0 rgba(76, 175, 80, 0); }
}
.result-highlight {
  animation: result-highlight 0.6s ease-out;
}
```

Track previous top 5 hashes. When a new hash appears that wasn't in the previous set, apply the animation class (remove after animation ends).

**Part H: Progress bar with timer**

The progress bar should also be updated via a `setInterval` on the main thread (every 200ms) independent of worker messages, so it animates smoothly even between worker progress messages:
- Start interval when search begins
- Calculate elapsed from `searchStartTime`
- Update bar width: `(elapsed / timeout) * 100`%
- Update time text: `{elapsedSeconds}s / {totalSeconds}s`
- Clear interval when COMPLETE received or stop clicked

**Part I: Accumulation across runs**

- Pass `currentTopResults` to the worker as `existingTopResults`
- Worker merges existing with new candidates
- Summary line updates with cumulative stats
- Track `totalAttempts` and `totalDecodableCount` across runs

- Top results area stays hidden or shows "No results found"
