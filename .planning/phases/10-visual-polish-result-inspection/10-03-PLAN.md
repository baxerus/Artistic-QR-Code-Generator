  <name>Task 1: Disable anti-aliasing in overlay canvas rendering</name>
  <files>index.html</files>
  <action>
In the `renderErrorOverlay` function (around line 15855), add `ctx.imageSmoothingEnabled = false` immediately after getting the canvas context.

Current code:
```javascript
const ctx = overlayCanvas.getContext("2d");
ctx.clearRect(0, 0, moduleCount, moduleCount);
```

Change to:
```javascript
const ctx = overlayCanvas.getContext("2d");
ctx.imageSmoothingEnabled = false;
ctx.clearRect(0, 0, moduleCount, moduleCount);
```

Root cause: The overlay canvas has anti-aliasing enabled by default. When drawing 1x1 pixel fillRect operations at integer coordinates on a 21x21 canvas that gets CSS-scaled to 120px+, the canvas anti-aliasing causes sub-pixel blending at module boundaries, creating the "blur" and "different shades of blue" effect.

This matches the pattern used in `renderQRToCanvas` (line 13068) which already sets `ctx.imageSmoothingEnabled = false`.

<task type="auto">
  <name>Task 2: Increase dashed border line width and dash pattern for visibility</name>
  <files>index.html</files>
  <action>
In the `renderProtectedBordersOverlay` function (around line 15896-15897), increase the line width and dash pattern:

Current code:
```javascript
ctx.lineWidth = 0.5; // Thicker line for visible dashed borders
ctx.setLineDash([2, 1]); // Larger dash pattern for visibility
```

Change to:
```javascript
ctx.lineWidth = 1; // Full pixel width for visibility when CSS-scaled
ctx.setLineDash([0.5, 0.25]); // Sub-module dashes visible after CSS scaling
```

Also add `ctx.imageSmoothingEnabled = false` at the start of the function to ensure crisp stroke rendering:

```javascript
function renderProtectedBordersOverlay(canvas, moduleCount, functionMask) {
  const ctx = canvas.getContext("2d");
  ctx.imageSmoothingEnabled = false;
  ctx.strokeStyle = PROTECTED_OUTLINE_COLOR;
  ctx.lineWidth = 1;
  ctx.setLineDash([0.5, 0.25]);
  // ... rest of function
```

Root cause: The previous values (lineWidth=0.5, dash=[2,1]) were intended for visibility but:
- lineWidth=0.5 is sub-pixel and gets anti-aliased to invisibility
- dash=[2,1] creates dashes spanning 2 full modules, which on a 7-module finder pattern edge creates only ~3 visible dash segments

The fix uses lineWidth=1 (full pixel, clearly visible after CSS scaling) and smaller dash fractions (0.5, 0.25) so dashes appear within each module edge segment.
