  <action>
Add a state persistence module in the main `<script>` block, immediately after the `const MAX_QR_VERSION = 10;` line (around line 13146) and the CAPACITIES_H_BYTE table. Place it before the `let qrCodeInstance = null;` line. This groups all persistence code together and makes the functions available to the rest of the app.

The module consists of:

**1. Storage key constants:**
```javascript
const STORAGE_KEYS = {
  URL: 'qr-art.url',
  VERSION: 'qr-art.version',
  PATTERN: 'qr-art.pattern'
};
```

**2. Debounce utility function:**
A standard debounce implementation that delays execution until `wait` ms after the last call. Used for URL input saves.
```javascript
function debounce(callback, wait) {
  let timeoutId = null;
  return (...args) => {
    window.clearTimeout(timeoutId);
    timeoutId = window.setTimeout(() => {
      callback.apply(null, args);
    }, wait);
  };
}
```

**3. Save functions (all wrapped in try-catch, silent on failure):**

- `saveURL(url)` — `localStorage.setItem(STORAGE_KEYS.URL, url)`. Simple string, no JSON.
- `saveVersion(version)` — `localStorage.setItem(STORAGE_KEYS.VERSION, version.toString())`. Convert to string.
- `savePattern(version, cells)` — Stores `JSON.stringify({ version: version, cells: Array.from(cells) })` to STORAGE_KEYS.PATTERN. The `Array.from()` handles the regular Array that PixelGrid uses.

All save functions: try-catch wrapping with `console.warn` on failure, no user-facing errors.

**4. Load functions (all wrapped in try-catch, return defaults on any failure):**

- `loadURL()` — Returns `localStorage.getItem(STORAGE_KEYS.URL) || ''`. Simple string.
- `loadVersion()` — Parses int from storage, validates `>= MIN_QR_VERSION && <= MAX_QR_VERSION`, returns `MIN_QR_VERSION` on invalid/missing.
- `loadPattern(expectedVersion)` — JSON.parse the stored pattern. Validate: (a) data is an object with `version` and `cells` array, (b) `data.version === expectedVersion`, (c) `data.cells.length === getModuleSize(data.version) ** 2`. Return `null` on any validation failure (version mismatch, size mismatch, corrupt JSON, missing key). Log warnings to console, never show to user.

**Important:** `loadPattern` returns `null` (not an empty array) when there is no valid pattern. The caller checks for null and skips restoration.

Place a `// ===== STATE PERSISTENCE =====` section header comment before the module.
