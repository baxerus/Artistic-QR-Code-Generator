# Requirements: Artistic QR Code Generator

**Defined:** 2026-02-10
**Core Value:** The painted pattern is sacred and never changes. The tool finds URL variants that naturally align with the art.

## v1.1 Requirements

Requirements for UX Overhaul & Optimization milestone. Each maps to roadmap phases.

### State Persistence

- [x] **PERS-01**: App state (URL, painted pattern, QR version, settings) persists across page reloads via localStorage
- [x] **PERS-02**: State restores correctly on page load, including re-rendering the QR code and pattern overlay

### Painting Overhaul

- [x] **PAINT-01**: Paint pattern rendered as overlay directly on top of QR code preview (not a separate canvas) ✓
- [x] **PAINT-02**: Protected QR areas (finder patterns, timing patterns, alignment patterns) visually marked as non-editable ✓
- [x] **PAINT-03**: Color selection via legend (black/white/unset) replaces click-to-cycle behavior ✓
- [x] **PAINT-04**: Left-click and drag paints selected color across multiple pixels ✓
- [x] **PAINT-05**: Right-click and drag paints opposite color (white<->black); does nothing when unset is selected ✓

### Generation Safety

- [x] **SAFE-01**: URLs ending in a hash fragment are detected and rejected with clear error message ✓
- [x] **SAFE-02**: When pattern exists, version dropdown change does NOT auto-generate QR code ✓
- [x] **SAFE-03**: When pattern exists, only "Generate QR Code" button triggers generation ✓
- [x] **SAFE-04**: When generating with same QR version, painted pattern is preserved on new QR code ✓
- [x] **SAFE-05**: When generating with changed QR version (dropdown or URL-forced), user is warned about pattern loss and must confirm ✓

### Optimization

- [x] **OPT-01**: Optimization auto-stops when 5 results with 0 errors are found ✓
- [x] **OPT-02**: Multiple Web Workers run optimization in parallel for faster search throughput ✓

### Configuration

- [x] **CONF-01**: QR version range expanded from 2-8 to 2-10 ✓
- [x] **CONF-02**: Maximum QR version is a named constant used everywhere, easy to change ✓

## Out of Scope

Explicitly excluded. Documented to prevent scope creep.

| Feature | Reason |
|---------|--------|
| QR error correction levels other than H | H provides maximum error tolerance needed for artistic patterns |
| Batch processing multiple URLs | Single URL workflow only |
| Server-side processing | Everything client-side in browser |
| QR versions beyond 10 | Sufficient range; max version is a constant if adjustment needed |
| Real-time camera scan testing | Error count is sufficient proxy for scannability |
